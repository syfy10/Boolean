BOOLEAN PRIVACY POLICY
Boolean - Version 1.2, July 17, 2026

THE SHORT VERSION
Boolean's local mode runs entirely on your computer. No telemetry, no
analytics, no tracking, no crash reports. Your code, prompts, and files stay
on your machine unless YOU choose a feature that needs the internet, such as
cloud AI, Google Sign-In, model downloads, web research, image generation, or
installation tools. Connecting a remote MCP server, HTTP agent, API connector,
or sending a screenshot, page, note, or project file to an online provider also
uses the network. Scheduled online actions use the network when they run. For
signed-in Boolean administrators, sync is enabled so notes and safe app
preferences can follow the same admin account across computers. API keys and
connector tokens remain local unless Boolean later adds a separate encrypted
key-vault feature.

WHAT STAYS ON YOUR COMPUTER
- Your conversations with the AI
- AI model files stored in the .saz\models folder in your user profile
- Any bundled starter model included with the installer
- Your settings, including any API keys you enter, stored at
  .saz\config.json in your user profile; protect this file like a password.
  Safe preferences such as theme, layout, notepad color, and browser state may
  sync for signed-in administrators, but API keys do not.
- Remote MCP server addresses and any Bearer tokens you save; connector
  secrets are stored locally and are not returned by Boolean's state API
- HTTP-agent and API connector addresses and credentials you save; protect
  these credentials like passwords
- Optional Gmail and Outlook OAuth client IDs, access/refresh tokens, and the
  connected account label. These stay in .saz\config.json on your PC and are
  not sent to Boolean's account service.
- Your Boolean account session token after Google Sign-In, stored locally so the
  app can stay signed in; sign out to remove it
- Your notepad content, except when signed in as an administrator with admin
  sync enabled
- Everything the agent reads or writes at your direction
- Task objectives, plans, debugging evidence, tool results, checkpoints,
  schedules, selected models, and approval settings used to continue or
  automate work
- Generated documents, spreadsheets, presentations, PDFs, images, and other
  local artifacts unless you send them to an online service
- A local Windows System Actions audit log at .saz\system-actions.jsonl,
  containing action names, time, result, and limited non-secret details such as
  a package ID or Settings page

WHAT BOOLEAN RECEIVES
When you use only local models, Boolean receives nothing and works with the
network cable unplugged.

If you choose optional Boolean account sign-in, Boolean's account service may
receive and store the minimum identity and session data needed for the account:
- Google account identifier, email, display name, and profile image
- session records used to keep you signed in
- for signed-in administrators, synchronized notepad titles, note content,
  embedded note images, timestamps, deletion records, and safe UI preferences
  such as theme, layout, notepad color, and browser state

WHEN DATA DOES LEAVE YOUR MACHINE
Only in these cases, and only because you set them up:
1. Cloud providers: if you add a provider API key and select that provider,
   your prompts and relevant context are sent to that provider's servers,
   governed by that provider's privacy policy.
2. Boolean account: Google Sign-In and session requests are sent to Boolean's
   account service. AI prompts are not routed through account sign-in.
3. Model downloads: downloading a model contacts the hosting service, such as
   Hugging Face, to fetch the file. Importing a model from a local drive
   contacts nothing.
4. Web browsing/research: if you ask Boolean to browse, search, or verify
   current information, requests, queries, and selected page content are sent
   to the sites, search engines, or research services involved.
5. Software installation helpers: if you ask Boolean to install software, tools
   such as winget contact their own servers.
6. Software updates: Boolean periodically requests a small update manifest from
   the saz3 Labs GitHub repository. If an update is available, Boolean may
   download its installer from an approved GitHub release URL and verify its
   SHA-256 hash before installation.
7. Remote MCP, HTTP-agent, and API connectors: when you enable a connection,
   Boolean sends protocol requests, relevant task content, tool arguments, and
   configured credentials to that service. Its privacy policy and terms apply.
8. Screenshots, browser pages, notes, and project files: these remain local when
   used with a local model. If you ask an online model or remote connector to
   inspect them, the selected content is sent to that provider or connector.
9. Optional Gmail or Outlook connectors: when you connect an account, OAuth and
   mail requests go directly from Boolean on your PC to Google or Microsoft.
   Boolean may read the messages you request and create drafts in that account.
   Sending is disabled while Draft-only is on and otherwise always requires a
   separate confirmation. Mail tokens and message contents are not routed
   through Boolean's Cloudflare account service.
10. Transactional messages: saz3 Labs may use Cloudflare Email Service to send
   Boolean account, security, support, or service notices. Cloudflare receives
   the recipient address and message data required to deliver those notices.
11. Scheduled actions: online AI tasks and connector calls send their configured
   task and relevant context when due. Webhooks receive the payload you
   configured, and opening a URL contacts that site. Local commands remain on
   this PC unless the command itself uses the network.
12. Image generation: prompts, source images, masks, and related instructions
   are sent to the online image provider you select. Generated or edited files
   remain local unless you send them elsewhere.
13. Admin sync: when a Boolean administrator is signed in, Boolean sends the
   administrator's notepad and safe-preferences snapshot to the Boolean account
   service over HTTPS and stores it in the account database. This allows notes,
   layout, theme, notepad color, and browser state to merge across computers
   signed in to that admin account. API keys and connector tokens are not
   synced. Local saving continues if sync is unavailable. Embedded images count
   toward the sync size limit. Signing out stops synchronization but does not
   erase local notes.

THIRD-PARTY PROVIDERS AND CONNECTIONS
OpenAI, Anthropic, Z.AI, Hugging Face, image providers, MCP/HTTP-agent/API
connector operators, websites, search engines, package repositories, GitHub,
Google, Cloudflare, and Stripe are independent services. Their own terms,
retention rules, charges, and privacy policies apply. Z.AI's Coding Plan uses
its separate coding endpoint and may be used only when Z.AI has approved
Boolean or otherwise supports the user's use. Boolean does not represent that
every subscription permits third-party use.

SOFTWARE UPDATE DATA
Update checks may reveal standard connection information, including your IP
address, app version, and user-agent, to GitHub or its delivery infrastructure.
Update checks do not upload your chats, prompts, files, notepad content, API
keys, or local model data. Admin sync is a separate account feature and is not
part of update checking. Before installing an update after the app
closes, Boolean backs up key local settings and chat data on your computer.

THIRD-PARTY CLOUD USAGE
Cloud AI requests use the provider API key selected by the user. Each provider
controls its own billing, quotas, retention, and acceptable-use terms. Boolean
does not grant free AI tokens or sell token packs in this version.

SOFTWARE NEEDED TO RUN BOOLEAN
The standard Windows install is designed to be self-contained. Boolean bundles
its app runtime, embedded Node.js backend, llama.cpp CPU engine, and required
app-local runtime DLLs. You do not need to install Node.js, .NET, Ollama,
Python, or the Microsoft Visual C++ Redistributable for the standard local app.

Boolean uses Microsoft Edge WebView2 for the app window and built-in browser.
Most Windows 11 PCs already include it. If WebView2 is missing, Windows or the
installer/store may need to install the WebView2 Runtime.

MINIMUM SYSTEM REQUIREMENTS
- Cloud mode: 4 GB RAM, internet connection, WebView2, and a supported
  third-party provider API key.
- Lightest local LLM mode: 8 GB RAM recommended, 2 GB free disk space, and a
  CPU with AVX2 preferred.
- Recommended local agent use: 16 GB RAM and 20 GB free disk space for models,
  project files, downloads, and update staging.
- Starter model mode: very small bundled models such as SmolLM2-135M can run
  on lower-end PCs, but answer quality and tool reliability are limited.
