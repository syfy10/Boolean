BOOLEAN PRIVACY POLICY
Boolean - Version 1.1, July 2026

THE SHORT VERSION
Boolean's local mode runs entirely on your computer. No telemetry, no
analytics, no tracking, no crash reports. Your code, prompts, and files stay
on your machine unless YOU choose a feature that needs the internet, such as
cloud AI, Google Sign-In, model downloads, web browsing, or installation tools.
Connecting a remote MCP server or sending a screenshot, page, note, or project
file to an online provider also uses the network.

WHAT STAYS ON YOUR COMPUTER
- Your conversations with the AI
- AI model files stored in the .saz\models folder in your user profile
- Any bundled starter model included with the installer
- Your settings, including any API keys you enter, stored at
  .saz\config.json in your user profile; protect this file like a password
- Remote MCP server addresses and any Bearer tokens you save; connector
  secrets are stored locally and are not returned by Boolean's state API
- Your Boolean account session token after Google Sign-In, stored locally so the
  app can stay signed in; sign out to remove it
- Everything the agent reads or writes at your direction
- A local Windows System Actions audit log at .saz\system-actions.jsonl,
  containing action names, time, result, and limited non-secret details such as
  a package ID or Settings page

WHAT BOOLEAN RECEIVES
When you use only local models, Boolean receives nothing and works with the
network cable unplugged.

If you choose optional Boolean account sign-in, Boolean's account service may
receive and store the minimum identity and session data needed for the account:
- Google account identifier, email, display name, and profile image
- session records used to keep you signed in

WHEN DATA DOES LEAVE YOUR MACHINE
Only in these cases, and only because you set them up:
1. Cloud providers: if you add a provider API key and select that provider,
   your prompts and relevant context are sent to that provider's servers,
   governed by that provider's privacy policy.
2. Boolean account: Google Sign-In and session requests are sent to Boolean's
   account service. AI prompts are not routed through account sign-in.
3. Model downloads: downloading a model contacts the hosting service, such as
   Hugging Face, to fetch the file. Importing a model from a local drive
   contacts nothing.
4. Web browsing/search: if you ask Boolean to browse or search the web,
   requests are sent to the sites/search engines involved.
5. Software installation helpers: if you ask Boolean to install software, tools
   such as winget contact their own servers.
6. Software updates: Boolean periodically requests a small update manifest from
   the saz3 Labs GitHub repository. If an update is available, Boolean may
   download its installer from an approved GitHub release URL and verify its
   SHA-256 hash before installation.
7. Remote MCP servers: when you enable an MCP connection, Boolean sends protocol
   requests, relevant tool arguments, and the configured Bearer token to that
   server. The server operator's privacy policy and terms apply.
8. Screenshots, browser pages, notes, and project files: these remain local when
   used with a local model. If you ask an online model or remote connector to
   inspect them, the selected content is sent to that provider or connector.

THIRD-PARTY PROVIDERS AND CONNECTIONS
OpenAI, Anthropic, Z.AI, Hugging Face, MCP server operators, websites, search
engines, package repositories, GitHub, Google, Cloudflare, and Stripe are
independent services. Their own terms, retention rules, charges, and privacy
policies apply. Z.AI's Coding Plan uses its separate coding endpoint and may be
used only when Z.AI has approved Boolean or otherwise supports the user's use.
Boolean does not represent that every subscription permits third-party use.

SOFTWARE UPDATE DATA
Update checks may reveal standard connection information, including your IP
address, app version, and user-agent, to GitHub or its delivery infrastructure.
Update checks do not upload your chats, prompts, files, notepad content, API
keys, or local model data. Before installing an update after the app closes,
Boolean backs up key local settings and chat data on your computer.

THIRD-PARTY CLOUD USAGE
Cloud AI requests use the provider API key selected by the user. Each provider
controls its own billing, quotas, retention, and acceptable-use terms. Boolean
does not grant free AI tokens or sell token packs in this version.

SOFTWARE NEEDED TO RUN BOOLEAN
The standard Windows install is designed to be self-contained. Boolean bundles
its app runtime, embedded Node.js backend, llama.cpp CPU engine, and required
app-local runtime DLLs. You do not need to install Node.js, .NET, Ollama,
Python, or the Microsoft Visual C++ Redistributable for the standard local app.

Boolean uses Microsoft Edge WebView2 for the app window and built-in browser.
Most Windows 11 PCs already include it. If WebView2 is missing, Windows or the
installer/store may need to install the WebView2 Runtime.

MINIMUM SYSTEM REQUIREMENTS
- Cloud mode: 4 GB RAM, internet connection, WebView2, and a supported
  third-party provider API key.
- Lightest local LLM mode: 8 GB RAM recommended, 2 GB free disk space, and a
  CPU with AVX2 preferred.
- Recommended local agent use: 16 GB RAM and 20 GB free disk space for models,
  project files, downloads, and update staging.
- Starter model mode: very small bundled models such as SmolLM2-135M can run
  on lower-end PCs, but answer quality and tool reliability are limited.
