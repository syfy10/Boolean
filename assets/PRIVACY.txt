BOOLEAN PRIVACY POLICY
Boolean - Version 1.0, July 2026

THE SHORT VERSION
Boolean's local mode runs entirely on your computer. No telemetry, no
analytics, no tracking, no crash reports. Your code, prompts, and files stay
on your machine unless YOU choose a feature that needs the internet, such as
cloud AI, Google Sign-In, model downloads, web browsing, or installation tools.

WHAT STAYS ON YOUR COMPUTER
- Your conversations with the AI
- AI model files stored in the .saz\models folder in your user profile
- Any bundled starter model included with the installer
- Your settings, including any API keys you enter, stored at
  .saz\config.json in your user profile; protect this file like a password
- Your Boolean Cloud session token after Google Sign-In, stored locally so the
  app can stay signed in; sign out to remove it
- Everything the agent reads or writes at your direction
- A local Windows System Actions audit log at .saz\system-actions.jsonl,
  containing action names, time, result, and limited non-secret details such as
  a package ID or Settings page

WHAT BOOLEAN RECEIVES
When you use only local models, Boolean receives nothing and works with the
network cable unplugged.

If you choose optional Boolean Cloud features, Boolean's backend may receive
and store the minimum account and billing data needed to provide the service:
- Google account identifier, email, display name, and profile image
- session records used to keep you signed in
- token balance, token ledger, daily usage counters, and free-grant expiration
- Stripe customer/payment/subscription status if paid billing is enabled
- cloud AI requests and usage metadata needed to answer, bill, prevent abuse,
  and troubleshoot the service

WHEN DATA DOES LEAVE YOUR MACHINE
Only in these cases, and only because you set them up:
1. Cloud providers: if you add a provider API key and select that provider,
   your prompts and relevant context are sent to that provider's servers,
   governed by that provider's privacy policy.
2. Boolean Cloud: if you sign in with Google and use cloud tokens, your account
   information, prompts, relevant context, and usage counts may be sent to
   Boolean's backend and then to the selected cloud model provider.
3. Model downloads: downloading a model contacts the hosting service, such as
   Hugging Face, to fetch the file. Importing a model from a local drive
   contacts nothing.
4. Web browsing/search: if you ask Boolean to browse or search the web,
   requests are sent to the sites/search engines involved.
5. Software installation helpers: if you ask Boolean to install software, tools
   such as winget contact their own servers.

FREE TOKENS AND USAGE LIMITS
The first 1,000 new Google Sign-In users may receive a free cloud token grant,
currently 100,000 tokens for the first 30 days with a 10,000 token daily cap.
After the first 1,000 new signups, new accounts may start with no free tokens.
Token use is metered and stored in the Boolean Cloud backend. For now, Boolean
Cloud counts one word as one token. Provider usage may differ from this
customer-facing meter. Free token amounts, daily caps, expiration, model
access, and availability may change. The free cloud tier may use GLM-4.7-Flash
through Cloudflare Workers AI or another low-cost cloud model selected by
Boolean.

PAYMENTS
If paid plans or token packs are enabled, payments are processed by Stripe.
Boolean may store Stripe customer IDs, subscription/payment status, and related
billing metadata, but Boolean should not store full card numbers.

SOFTWARE NEEDED TO RUN BOOLEAN
The standard Windows install is designed to be self-contained. Boolean bundles
its app runtime, embedded Node.js backend, llama.cpp CPU engine, and required
app-local runtime DLLs. You do not need to install Node.js, .NET, Ollama,
Python, or the Microsoft Visual C++ Redistributable for the standard local app.

Boolean uses Microsoft Edge WebView2 for the app window and built-in browser.
Most Windows 11 PCs already include it. If WebView2 is missing, Windows or the
installer/store may need to install the WebView2 Runtime.

MINIMUM SYSTEM REQUIREMENTS
- Cloud/online mode: 4 GB RAM, internet connection, WebView2, and a provider
  API key or Boolean Cloud account.
- Lightest local LLM mode: 8 GB RAM recommended, 2 GB free disk space, and a
  CPU with AVX2 preferred.
- Starter model mode: very small bundled models such as SmolLM2-135M can run
  on lower-end PCs, but answer quality and tool reliability are limited.

CHILDREN
Boolean is a developer tool and is not directed at children under 13.

CHANGES
Future versions of this policy ship with the software itself and may also be
published with the Boolean Cloud backend when cloud account features are used.
